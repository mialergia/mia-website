Proyecto

En base a los requerimientos planteados por parte de los integrantes de la facultad de ciencias y los médicos del Hospital Pasteur, se decidió desarrollar una aplicación que le permitiera a los usuarios conocer los niveles de polen y esporas, de árboles, hierbas y hongos, asi cómo también tener un diario de síntomas. Para almacenar tanto los datos de pólenes y esporas, asi como también los nuevos síntomas dados de alta por parte de los usuarios fue necesario utilizar una base de datos en conjunto con un servidor para procesar los datos almacenados.

Backend

Para la implementación del backend utilizamos Django. Django es un framework basado en python, libre y open-source. El backed provee una api web, utilizada para registrar usuarios, obtener el último reporte de polen, dar de alta nuevos síntomas, entre otras funciones. También provee una admin page, la misma es utilizada entre otras cosas, para dar de alta nuevos reportes de polen.
Para el almacenamiento de los datos se utilizó una base de datos PostgreSQL. PostgreSQL es una poderosa base de datos relacional, open-source, con más de 30 años de desarrollo activo, que le ha dado una sólida reputación por su confiabilidad, robustez de funciones y rendimiento.

Para acceder al código del backend visite: https://github.com/nataliacampiglia/mia-backend

Frontend

Para el frontend decidimos utilizar React Native, el mismo es un framework open-source, que combina el desarrollo nativo de los sistemas operativos de iOS y Android en simultáneo, junto con React, una librería de Javascriptespecialmente pensada para crear interfaz de usuario.

Para acceder al código del frontend visite: https://github.com/tamarasuarezrod/mia-react-native
